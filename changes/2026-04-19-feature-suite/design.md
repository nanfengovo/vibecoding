# Request
用户提出 5 个需求：
1. 股票详情页周期选择后增加“年（Y）”和“自由时间范围”，并修复从股票详情页跳转其他页面时出现的 bug。
2. 首页仪表盘不再使用虚拟数据，改为展示真实关注股票并提供类似多股看板的实时视图。
3. 前端支持明暗主题切换。
4. 增加 AI 分析功能，并可在系统配置中配置 OpenAI API Key。
5. 增加低代码平台，用于编辑监控规则、交易下单、回测任务等模块。

# Current Behavior
- 股票详情页仅支持 `1/5/15/60/D/W/M` 周期，没有 `Y` 和自定义时间范围。
- 股票详情页存在重复刷新触发，路由切换时可能出现多次请求或状态闪烁。
- Dashboard 使用随机模拟资产曲线与固定市场指数，非真实监控数据。
- 系统配置没有 OpenAI 配置项，也没有 AI 分析接口/页面。
- 无“低代码平台”入口，规则/交易/回测只能分散在已有页面操作。
- 整体 UI 为固定浅色主题。

# Proposed Change
1. **股票详情增强**
- 新增 `Y` 周期按钮。
- 新增“自定义范围”日期选择（`datetimerange`），选择后按时间范围拉取 K 线并更新图表。
- 修复股票详情页路由切换 bug：避免 `watch + onMounted` 双触发重复刷新；统一单一路径触发并增加请求并发保护。

2. **Dashboard 真实化**
- 去掉随机资产曲线和固定指数虚拟数据。
- 使用真实关注列表行情构建“多股看板”（卡片网格 + 小型走势图/关键指标）。
- 保留交易、策略等现有能力，但数据来源改为实时接口和关注列表。

3. **主题切换**
- 增加全局主题状态（`light/dark`），支持本地持久化。
- 在主布局头部加入主题切换按钮。
- 使用 CSS 变量 + `html.dark` 覆盖基础色，保持现有视觉结构并兼容 Element Plus 深色变量。

4. **AI 分析功能**
- 配置中心新增 `OpenAI` 分组（API Key、Base URL、Model、启用开关）。
- 后端新增 AI 分析接口：接收股票上下文（行情、K 线摘要），调用 OpenAI 生成分析结果（趋势、风险、操作建议）。
- 股票详情页新增“AI 分析”卡片，可一键生成与刷新分析。

5. **低代码平台**
- 新增独立页面“低代码工作台”。
- 提供 3 类模块编辑器（监控规则、交易下单、回测任务），以 JSON Schema + 可视表单方式组合。
- 支持“预览请求体 + 一键执行调用现有 API”。
- 支持模板保存到本地（localStorage）和快速复用。

# Open Questions
- OpenAI 模型默认值选型：优先兼容性，默认 `gpt-4o-mini`，并允许用户自定义。
- AI 分析是否持久化：当前阶段先不落库，按需实时生成。
- 低代码编排是否支持跨模块流程：当前先做单模块执行与模板复用，后续可扩展流程编排。

# Decision
- 采用“最小侵入式改造”：尽量复用现有 Controller 和 API，不破坏当前交易/策略/回测主流程。
- 主题采用 CSS 变量方案，不大规模改写每个页面样式。
- AI 调用集中在后端，前端不直接暴露 API Key。
- 低代码平台先实现“编辑 + 调用 + 保存模板”，不引入额外重型依赖。

# Risks
- 部分接口字段历史上存在命名不一致（如 `current`/`price`），需要前端兼容映射。
- OpenAI 网络与限额可能导致分析请求失败，需要清晰错误提示与降级。
- 深色主题切换可能影响个别 scoped 样式，需要补丁覆盖。
- 大量前端改动需要重点验证路由切换与响应性能。

# Validation Plan
1. 手动验证股票详情页：
- 周期切换 `D/W/M/Y` 正常；
- 自定义时间范围生效；
- 从股票详情跳转到其他页面无异常请求风暴或 UI 抖动。
2. 手动验证 Dashboard：
- 无随机数据；
- 关注股票看板实时显示。
3. 手动验证主题：
- 明暗切换即时生效，刷新后保持。
4. 手动验证 AI：
- 在设置页保存 OpenAI 配置；
- 详情页点击“AI 分析”可返回文本结果。
5. 手动验证低代码平台：
- 规则/交易/回测三种模板可编辑、执行、保存、复用。
