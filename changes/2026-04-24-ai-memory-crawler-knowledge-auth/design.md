# Request

实现带长期记忆的 AI、可配置信息采集模块、Markdown 知识库、独立知识库 AI，以及账号权限和用户数据隔离。

# Current Behavior

当前系统已有长桥行情、AI 分析、多模型配置和前端 AI Chat 历史。AI Chat 历史主要保存在浏览器本地，后端没有用户、权限、长期记忆、知识库或爬虫实体。系统配置保存在 `SystemConfigs`，包含长桥和 AI 模型配置，必须保持兼容并避免覆盖既有密钥。

# Proposed Change

新增账号密码登录和 JWT 鉴权，使用默认管理员承接历史数据。新增 AI 会话、消息、长期记忆、采集源、采集文档、知识库、知识库文档和分片等实体。AI Chat 在调用模型前注入用户长期记忆、会话上下文和可选知识库片段；爬虫模块优先抓取长桥 Markdown 内容，也支持白名单外部 URL/RSS；知识库支持 Markdown 导入、导出、检索和独立问答。

# Open Questions

第一版不引入向量数据库，知识库检索使用数据库文本字段和关键词得分。后续如需要更强语义检索，再补充 embedding 和向量索引。

# Decision

采用账号密码 + JWT。系统级长桥/OpenAI 配置继续全局共享，不按用户隔离；用户隔离覆盖业务数据、AI 会话、长期记忆、爬虫源和知识库。数据库迁移采用启动时幂等 schema 升级，避免 `EnsureCreated` 无法更新已有线上库的问题。

补全实现时统一采用“默认需要登录”的后端策略：公共行情查询保持可访问，关注列表、策略、交易、回测、复盘、监控、AI、爬虫、知识库均要求 JWT；系统配置和用户管理仅管理员可访问。前端新增登录页和 Axios Token 注入，AI Chat 会话来源迁移到后端，`localStorage` 只保留草稿和兼容未登录前的临时状态。爬虫定时任务使用 Quartz 每分钟扫描启用源并按 `CrawlIntervalMinutes` 执行。

# Risks

本次改动跨度较大，风险集中在认证启用后前端请求头、数据库增量建表、旧数据绑定默认管理员、AI 上下文过长和爬虫外部来源失败。配置风险通过不清空 `SystemConfigs`、保留 masked secret 语义和只新增表/列来控制。

已有 `AdminMigrationController` 中存在清空导入逻辑，与本需求的安全约束冲突；本次将移除破坏性导入能力或改为拒绝执行，只保留只读摘要能力，避免误触清空 `SystemConfigs`。

# Validation Plan

验证后端构建、前端构建、登录接口、配置读取不丢失、AI Chat 会话持久化、记忆隔离、知识库 Markdown 导入导出、长桥 Markdown 抓取和未登录保护接口返回 401。
