# Request
用户要求本轮完成以下内容：
1. 修复明暗切换异常。
2. 修复关注列表显示异常。
3. 修复监控规则无法编辑。
4. 修复交易记录页面表格对齐和筛选不生效。
5. 扩展低代码模块：支持多规则配置、接口查询/交易配置、逻辑处理（含公式与条件）、自动交易、拖拽排序、模板导出和执行开关。
6. 支持移动端显示。
7. 推送到指定 GitHub 仓库并部署到 Streamlit，数据库使用免费方案。

# Current Behavior
- 明暗切换后仍存在局部硬编码亮色背景和文本，可读性差。
- 关注列表在部分行情字段来源不一致时显示 `-`。
- 监控规则列表“编辑”逻辑之前未完整实现（已部分修复，需继续验证）。
- 交易记录只按部分字段过滤，状态和日期筛选未完整生效；部分数值字段未规范化导致显示和对齐问题。
- 低代码模块目前是单模块参数表单，缺少拖拽式规则流、公式参数、执行开关、导入导出模板。
- 移动端没有系统化响应式处理。

# Proposed Change
- 统一主题变量使用，替换布局中的硬编码颜色并补充移动端样式断点。
- 在关注列表继续使用行情字段归一化与兜底显示逻辑。
- 完善监控规则编辑流程与表单回填/重置。
- 在交易 API 侧增加交易数据规范化和前端筛选参数支持（symbol/side/status/date），并在页面中联动筛选触发刷新。
- 将低代码模块升级为“规则流编排”：
  - 支持多个步骤（查询、条件、公式、交易、通知）；
  - 支持拖拽排序；
  - 支持按规则启用/禁用；
  - 支持执行时依据逻辑自动下单；
  - 支持模板导入/导出 JSON。
- 增加页面级响应式适配（头部、筛选区、表格横向滚动、侧边栏行为）。
- 补全实时链路：
  - 后端统一 SignalR 推送服务；
  - 监控/交易/行情刷新时主动推送；
  - 前端断线重连后自动重订阅，避免“连接恢复但数据不更新”。

# Open Questions
- Streamlit 目标与当前前后端技术栈不一致（当前是 Vue + .NET API）。部署可能需新增 Streamlit 壳层或改为静态/容器部署方案，需在交付时明确说明可行路径和限制。
- 推送到指定仓库需要本地 git remote/权限是否可用，若不可用需用户提供凭证或手动操作。

# Decision
- 先优先完成本地可验证的功能修复与增强（1/2/3/4/5/7）。
- 推送与部署部分在功能完成后执行可行性检查；若环境受限，提供可直接执行的命令与替代部署方案。

# Risks
- 低代码规则流涉及自动交易，若参数校验不严会误触发交易。
- 交易列表数据来自本地+远端合并，字段命名差异可能导致边界数据展示异常。
- 移动端适配若只做局部样式，可能遗漏个别页面组件。

# Validation Plan
- 前端执行构建并检查关键页面：
  - 主题切换（浅/深）；
  - 关注列表价格字段；
  - 监控规则编辑/保存；
  - 交易筛选（状态/方向/关键词/日期）；
  - 低代码规则拖拽、执行开关、导出导入；
  - 移动端断点下布局。
